## 2. Как оценивается качество

Три типа критериев:

1. **Детерминированные проверки** (`scorers/deterministic.py`) — инструменты/агенты,
   факты в ответе, `success_criteria`.
2. **Прокси retrieval** (`rag_eval.py`) — recall@k / precision@k / MRR.
3. **Абляции** — критик on/off, аугментации запроса, baseline без инструментов.

Светофор: наихудший из тест → раздел → блок → итог. 

### Пороги 

| Параметр | Значение |
|---|---|
| pass-rate 🟢 GREEN | ≥ 0.85 |
| pass-rate 🟡 YELLOW | ≥ 0.70 |
| pass-rate 🔴 RED | < 0.70 |
| Мин. значимое улучшение (альтмоделирование) | ≥ 3 п.п. |
| Milestones / эталоны | не требуются |

## 4. RAG: качество retrieval

RAG обслуживает Compliance Agent (визы, багаж, правила перевозчиков).

**Стек** (`app/rag.py`, тот же код в оценке и в проде):
`BAAI/bge-m3` (нормализованные) → pre-filtered HNSW в Qdrant
(`airline_BAAI_bge-m3`), фильтр `doc_type`/`entity` **до** векторного поиска.
Прод-`top_k` = 3, в оценке `top_k` = 5.

| Метрика | Значение | Как считается |
|---|---|---|
| recall@k | **0.975** | доля кейсов с релевантным документом в top-k |
| precision@k | **0.385** | доля релевантных среди k выданных |
| MRR | **0.917** | средний обратный ранг первого релевантного |

- **recall@5 = 0.975**: 39/40; MRR 0.917 — релевантный документ чаще всего **первый**.
- **precision@k = 0.385 — by design.** Обычно 1–3 релевантных чанка из 5
  (`n_relevant=x/5`); precision@5 структурно ограничен. KPI домена — recall@k и MRR.
- **Провал:** `rag_carrier_s7` (recall=0). Остальные перевозчики — recall 1.0
  (например `rag_carrier_aeroflot`, precision 1.0). Точечное покрытие/разметка S7,
  не системная деградация.
